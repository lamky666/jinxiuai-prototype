API 快速开始

5 分钟接入，与 OpenAI SDK 完全兼容。只需改一行 baseURL，所有 OpenAI 生态工具开箱即用。

📌 核心原则：锦绣AI 提供 OpenAI 兼容协议，任何支持自定义 baseURL 的 OpenAI 客户端工具，都能直接对接。请在工具的配置文件中把 baseURL 设为 https://api.jinxiu.cn/v1，把 API Key 设为你在锦绣控制台生成的 Key，模型名按本页对应章节填入即可。

1

注册并获取 API Key

注册后进入 控制台，创建 API Key。格式为 jinx_sk_xxxxxxxxxxxx。Key 创建时仅展示一次，请妥善保存。

2

发送第一个请求

把你的 OpenAI 代码中的 baseURL 改为锦绣AI地址即可。

curl

# 只需修改 baseURL 和 model 名称
curl https://api.jinxiu.cn/v1/chat/completions \
-H "Authorization: Bearer jinx_sk_YOUR_KEY" \
-H "Content-Type: application/json" \
-d '{"model":"deepseek-chat","messages":[{"role":"user","content":"你好"}]}'

Python

# pip install openai
from openai import OpenAI
 
client = OpenAI(
api_key="jinx_sk_YOUR_KEY",
base_url="https://api.jinxiu.cn/v1"
)
 
response = client.chat.completions.create(
model="deepseek-chat",
messages=[{"role": "user", "content": "你好"}]
)
print(response.choices[0].message.content)

Node.js

// npm install openai
import OpenAI from 'openai';
 
const client = new OpenAI({
apiKey: 'jinx_sk_YOUR_KEY',
baseURL: 'https://api.jinxiu.cn/v1',
});
 
const response = await client.chat.completions.create({
model: 'deepseek-chat',
messages: [{ role: 'user', content: '你好' }],
});
console.log(response.choices[0].message.content);
3

验证模型透明性

每次调用后，检查响应头即可验证实际模型。

响应头示例：

x-model-actual: deepseek-chat-v2
x-token-usage: input=12,output=48
x-cost: 0.0006
x-balance-remaining: 4.87
x-request-id: req_c8f3a2b1

认证与 API Key

所有 API 请求都需要在 Authorization 头中携带 Bearer Token，Token 即你在控制台创建的 API Key。

⚠️ 安全提示

API Key 拥有你账户的完整访问权限。请勿提交到 Git、客户端代码或公网可访问的页面。若怀疑泄露，请立即在控制台禁用并重建。

✓
环境变量：生产环境请用 JINX_API_KEY 环境变量存储，不要硬编码。
✓
分组：可创建多个 Key 并分配到不同组（开发/测试/生产），按组设置速率上限。
✓
余额预警：余额低于阈值时，自动通过短信/邮件提醒，可在控制台配置。

Codex CLI

OpenAI

OpenAI 官方的命令行 Agent，用于在终端中直接用自然语言驱动代码生成、文件编辑、命令执行。

环境变量配置（推荐）

# macOS / Linux（写入 ~/.zshrc 或 ~/.bashrc）
export OPENAI_API_KEY="jinx_sk_YOUR_KEY"
export OPENAI_BASE_URL="https://api.jinxiu.cn/v1"
 
# Windows PowerShell
$env:OPENAI_API_KEY="jinx_sk_YOUR_KEY"
$env:OPENAI_BASE_URL="https://api.jinxiu.cn/v1"

运行示例

# 启动交互模式
codex
 
# 单次任务模式
codex "给当前目录加一个 README.md"
 
# 指定模型（锦绣提供的 Claude 3.5 Sonnet）
codex --model gpt-4o "重构 utils.py"

Claude Code

Anthropic

Anthropic 官方的终端 Agent，基于 Claude 3.5 Sonnet，专为代码场景优化。需要在 ~/.claude.json 或环境变量中覆盖默认网关。

# ~/.claude.json
{
"apiKey": "jinx_sk_YOUR_KEY",
"baseURL": "https://api.jinxiu.cn/v1",
"model": "claude-3-haiku"
}
 
# 或通过环境变量
export ANTHROPIC_API_KEY="jinx_sk_YOUR_KEY"
export ANTHROPIC_BASE_URL="https://api.jinxiu.cn/v1"

💡 提示：Claude Code 默认走 Anthropic 协议，锦绣已自动把 OpenAI 格式转换为 Anthropic 格式，对业务方完全透明。

WorkBuddy

自家

WorkBuddy 是我们自家的桌面端 AI 助手，支持代码生成、文件问答、命令执行、浏览器自动化。在「设置 → 模型服务」中选择「自定义 OpenAI 兼容」即可。

配置项：

  • • 服务类型：OpenAI 兼容
  • • Base URL：https://api.jinxiu.cn/v1
  • • API Key：jinx_sk_...
  • • 默认模型：gpt-4o（也可选 deepseek-chat / claude-3-haiku）

配置文件位置（高级用户）

# macOS
~/Library/Application Support/WorkBuddy/config.json
 
# Windows
%APPDATA%\WorkBuddy\config.json
 
# Linux
~/.config/workbuddy/config.json

Cursor

IDE

AI-first 代码编辑器，Settings → Models → OpenAI API Key 下方可填入自定义 Base URL。

# Cursor Settings → Models → OpenAI
Override OpenAI Base URL: https://api.jinxiu.cn/v1
OpenAI API Key: jinx_sk_YOUR_KEY
 
# 或在 ~/.cursor/config.json 写入
{
"openai.baseURL": "https://api.jinxiu.cn/v1",
"openai.apiKey": "jinx_sk_YOUR_KEY"
}

Cline

VSCode

VSCode 上的开源 AI Agent。安装 Cline 插件后，API Provider 选择「OpenAI Compatible」。

# Cline 配置（VSCode Settings）
cline.apiProvider: openai
cline.openAiBaseUrl: https://api.jinxiu.cn/v1
cline.openAiApiKey: jinx_sk_YOUR_KEY
cline.openAiModelId: gpt-4o

Continue

VSCode / JetBrains

开源的 AI 编程助手，支持 VSCode 和 JetBrains 全家桶。编辑 ~/.continue/config.json：

{
"models": [
{
"title": "锦绣 GPT-4o",
"provider": "openai",
"model": "gpt-4o",
"apiBase": "https://api.jinxiu.cn/v1",
"apiKey": "jinx_sk_YOUR_KEY"
}
]
}

ChatGPT 风格 Web

Web

开源的 ChatGPT Next Web、LibreChat、Lobe Chat 等项目，部署时通过环境变量指向锦绣网关即可。

# ChatGPT-Next-Web (.env.local)
OPENAI_API_KEY=jinx_sk_YOUR_KEY
BASE_URL=https://api.jinxiu.cn
DEFAULT_MODEL=gpt-4o
 
# LobeChat 环境变量
OPENAI_API_KEY=jinx_sk_YOUR_KEY
OPENAI_PROXY_URL=https://api.jinxiu.cn/v1

也可以直接使用 锦绣AI 内置的在线对话，无需部署。

Raycast AI

macOS

Raycast 的 AI 插件支持自定义 OpenAI 端点：Raycast Settings → AI → Advanced → Custom Provider。

# Raycast AI Custom Provider
Provider Name: 锦绣AI
API Key: jinx_sk_YOUR_KEY
Base URL: https://api.jinxiu.cn/v1
Model: gpt-4o

LangChain / LlamaIndex

两个最流行的 LLM 应用框架。只需在初始化时传入 base_url 与 api_key。

LangChain（Python）

from langchain_openai import ChatOpenAI
 
llm = ChatOpenAI(
model="gpt-4o",
api_key="jinx_sk_YOUR_KEY",
base_url="https://api.jinxiu.cn/v1"
)
llm.invoke("用一句话介绍 LangChain")

Dify / FastGPT

在「系统模型供应商」中添加「OpenAI 兼容」类型，填入网关地址和 Key 即可。

  • • 供应商：OpenAI Compatible
  • • API Key：jinx_sk_YOUR_KEY
  • • API Endpoint：https://api.jinxiu.cn/v1
  • • 模型名称：填入本页「可用模型」中的 ID

常见问题

如何确认没有被"偷换模型"？

每次 API 调用后，检查响应头中的 x-model-actual 字段。这个字段显示实际处理你请求的模型 ID，不会被伪造。

充值后余额会过期吗？

不会。余额永久有效，无有效期限制。按量计费，扣完即止。

我的工具不在上面的列表里，能用吗？

只要你的工具支持自定义 OpenAI 兼容端点（绝大多数现代 AI 工具都支持），就能用。把 baseURL 设为 https://api.jinxiu.cn/v1 即可，具体配置项名称可能叫 apiBase / endpoint / api_endpoint，请参照工具自身文档。
